# 三十年图形进化史：从像素方块到照片级真实

> 从 1992 年《Wolfenstein 3D》的伪 3D 走廊，到 2025 年《GTA VI》的照片级迈阿密——游戏渲染技术走过了一条波澜壮阔的进化之路。本文带你回顾每一个关键转折点。

## 引言：一部浓缩的技术革命史

如果把游戏渲染技术的三十年浓缩成一句话，那就是：

> **从"让玩家相信这是 3D"到"让玩家分不清这是游戏还是现实"。**

这条进化之路并不是线性的——每隔几年就会出现一次"范式跃迁"，彻底改变游戏画面的面貌。让我们沿着时间线，回顾那些改变历史的技术里程碑。

---

## 第一纪元：伪 3D 与软件渲染（1992–1996）

### 起点：Wolfenstein 3D（1992）

id Software 的 John Carmack 创造了游戏图形史上的第一个奇迹。《Wolfenstein 3D》使用**光线投射（Raycasting）** 技术——从玩家视角向前发射光线，计算光线与墙壁的交点来确定绘制高度。

严格来说，这不是真正的 3D——所有墙壁必须是垂直的，地板和天花板是纯色。但在 386 处理器上实现了流畅的第一人称视角，这本身就是一个工程奇迹。

```
技术要点：
• 2D 地图 + Raycasting = 伪 3D 效果
• 无浮点运算，纯整数定点数学
• 320×200 分辨率，~70 FPS
• 所有渲染在 CPU 上完成（软件渲染）
```

### 革命：DOOM（1993）

Carmack 的下一作引入了革命性的 **BSP 树（Binary Space Partitioning）** 场景管理和可变高度地板/天花板。虽然仍然不是真正的 3D（不支持上下看），但画面复杂度实现了数量级的提升。

### 真 3D 降临：Quake（1996）

《Quake》是游戏史上第一个完全实时 3D 渲染的 FPS 游戏。Carmack 实现了：

- **真正的多边形 3D 世界**（不再是 2D 地图的投射）
- **光照贴图（Lightmap）**：预计算的静态光照烘焙到纹理上
- **Z-Buffer**：正确处理物体前后遮挡关系

但这一切仍然是**纯 CPU 软件渲染**——没有 GPU 的参与。

> **时代特征**：一切渲染靠 CPU 硬算，程序员的数学和汇编功底决定画面上限。

---

## 第二纪元：GPU 革命与固定管线（1996–2001）

### 3dfx Voodoo：GPU 的诞生（1996）

3dfx 推出的 Voodoo 显卡开创了**硬件加速 3D 渲染**的时代。它能在硬件层面完成：
- 纹理映射（Texture Mapping）
- 双线性过滤（Bilinear Filtering）
- Z-Buffer 深度测试
- 雾效（Fog）

从此，CPU 不再需要逐像素绘制——GPU 接管了图形输出。

### 固定功能管线（Fixed-Function Pipeline）

早期 GPU 的渲染管线是**固定的**——开发者只能调整有限的参数（光源位置、材质颜色、雾的浓度），无法自定义渲染逻辑。

```
固定管线流程：
顶点变换 → 光照计算 → 光栅化 → 纹理采样 → 深度测试 → 输出
    ↑           ↑                    ↑
  硬件固定    硬件固定             硬件固定
  (不可编程)  (不可编程)          (不可编程)
```

典型的 API 是 **OpenGL 1.x** 和 **Direct3D 7**，开发者通过设置"渲染状态"来控制画面效果。

### 标志性游戏

- **《Half-Life》（1998）**：Source 引擎的前身 GoldSrc，展示了 GPU 加速的威力
- **《Unreal》（1998）**：Epic Games 的初代引擎，引入了彩色动态光照
- **《Quake III Arena》（1999）**：id Tech 3 引擎，曲面细分（Bezier Patches）和多纹理混合

> **时代特征**：GPU 接管渲染，但程序员只能在硬件给定的"菜单"里选择。

---

## 第三纪元：可编程着色器（2001–2007）

### 颠覆者：GeForce 3 与 Shader Model 1.0（2001）

NVIDIA 的 GeForce 3 首次引入**可编程顶点着色器和像素着色器**。这意味着开发者终于可以**自己编写代码**来控制每个顶点和每个像素的渲染逻辑。

这是一个划时代的转变——从"选择参数"变成了"编写程序"。

### DirectX 9 与 Shader Model 2.0/3.0（2002–2004）

Shader Model 2.0 和 3.0 大幅增加了着色器指令数量和功能。这一时期涌现了大量经典技术：

| 技术 | 原理 | 代表作 |
|------|------|--------|
| **法线贴图（Normal Mapping）** | 用纹理模拟微小的表面凹凸 | 《Doom 3》(2004) |
| **HDR 渲染** | 高动态范围光照 + 色调映射 | 《Half-Life 2: Lost Coast》(2005) |
| **阴影贴图（Shadow Mapping）** | 从光源视角渲染深度图 | 大量游戏采用 |
| **屏幕空间环境光遮蔽（SSAO）** | 近似计算环境遮挡 | 《Crysis》(2007) |
| **延迟渲染（Deferred Rendering）** | 先渲染几何信息（G-Buffer），再统一计算光照 | 《S.T.A.L.K.E.R.》(2007) |

### CryEngine 与"能跑 Crysis 吗？"

2007 年，Crytek 的《Crysis》成为了图形技术的新标杆。CryEngine 2 集成了当时几乎所有前沿渲染技术：
- 实时体积光（Volumetric Lighting）
- 动态植被系统
- 实时全局光照近似
- 大规模场景破坏

"Can it run Crysis?"（能跑《Crysis》吗？）成为了衡量硬件性能的流行语，至今仍被引用。

> **时代特征**：着色器可编程带来了技术爆发，每一年都有新的视觉效果被发明。

---

## 第四纪元：统一着色器与物理渲染（2007–2018）

### 统一着色器架构（2006–2007）

DirectX 10 引入了**统一着色器模型（Unified Shader Model）**。以前顶点着色器和像素着色器是独立的硬件单元，现在统一为通用计算单元——GPU 可以根据负载动态分配资源。

这为日后的 **Compute Shader** 和 **GPGPU 计算**奠定了基础。

### 物理渲染（PBR）革命（2012–2015）

2012 年，迪士尼发表了 **"Disney Principled BRDF"** 论文，提出了一套美术友好的物理渲染参数化模型。随后，这一理念迅速被整个游戏行业采纳：

- **《使命召唤：高级战争》（2014）**：较早采用 PBR 的 3A 游戏
- **虚幻引擎 4（2014）**：将 PBR 作为默认材质模型
- **《命运》（2014）**：Bungie 的 PBR 实现成为行业参考

PBR 的核心思想是：**用物理正确的数学模型描述光与材质的交互**。这意味着同一套材质参数在不同光照条件下都能产生正确的视觉效果，大幅减少了美术师的反复调参工作。

（详见本系列第三篇：[材质与光照的科学：PBR 如何统一了游戏画面](./03-材质与光照的科学：PBR如何统一了游戏画面.md)）

### 延迟渲染成为主流

这一时期，**延迟渲染（Deferred Rendering）** 和其变体 **贴图延迟渲染（Tiled Deferred）** 成为业界主流：

```
传统前向渲染：每个物体 × 每个光源 = 巨大开销
延迟渲染：    先渲染 G-Buffer → 后统一计算所有光照
```

这使得场景中可以出现**成百上千个动态光源**而不会崩溃——对开放世界游戏至关重要。

> **时代特征**：PBR 统一了材质标准，延迟渲染支撑了大规模场景，画面进入"看起来真实但还不够真实"的阶段。

## 第五纪元：光线追踪与次世代（2018–至今）

### RTX 发布：实时光线追踪元年（2018）

2018 年，NVIDIA 发布 GeForce RTX 2080，首次在消费级 GPU 上集成了**专用光线追踪硬件单元（RT Core）**。配合 **DirectX Raytracing（DXR）** API，实时光线追踪从学术梦想变成了商业现实。

第一批支持光追的游戏迅速涌现：
- **《战地V》（2018）**：光追反射
- **《控制》（2019）**：光追反射 + 全局光照 + 阴影
- **《我的世界 RTX》（2020）**：完整的路径追踪

### 虚幻引擎 5：Nanite + Lumen（2022）

Epic Games 的 UE5 带来了两项颠覆性技术：

- **Nanite**：虚拟化微多边形几何体，自动 LOD，支持影视级资产直接导入（详见第五篇）
- **Lumen**：软件 + 硬件混合的全局光照系统，无需烘焙（详见第四篇）

这标志着游戏引擎进入了**"影视级实时渲染"**的新阶段。

### AI 渲染的崛起

NVIDIA 的 **DLSS（Deep Learning Super Sampling）**、AMD 的 **FSR** 和 Intel 的 **XeSS** 将 AI 引入了渲染管线：

- 以低分辨率渲染，用 AI 网络超分到高分辨率
- DLSS 3 引入**帧生成（Frame Generation）**——AI 凭空生成中间帧
- DLSS 4 引入**多帧生成**——每渲染一帧生成三帧

这使得在 4K 分辨率下开启全光追变得可行。

（详见第六篇：[以假乱真的像素魔法：DLSS、FSR 与 AI 超分辨率](./06-以假乱真的像素魔法：DLSS、FSR与AI超分辨率.md)）

### 全路径追踪的曙光

最新的游戏开始支持**全路径追踪（Full Path Tracing）**——不再混合光栅化和光追，而是完全用光线追踪来渲染整个场景：

- **《赛博朋克2077: Overdrive》（2023）**：首个全路径追踪的开放世界 3A 游戏
- **《传送门 RTX》（2023）**：经典游戏的全路径追踪重制
- **《Alan Wake 2》（2023）**：堪称当代画面天花板

> **时代特征**：光线追踪从"噱头"变成"标配"，AI 辅助渲染成为新常态，游戏画面开始逼近电影 CG。

## 技术进化的三条主线

回顾三十年历程，可以清晰地看到三条相互交织的进化主线：

### 主线一：从近似到精确

```
Phong 着色（经验模型）
  → Cook-Torrance BRDF（物理模型）
    → 实时路径追踪（精确模拟光传输方程）
```

渲染技术的核心追求始终是：**更精确地模拟光的行为**。每一次范式跃迁，都是在用更少的近似来逼近光传输方程（Rendering Equation）的精确解。

### 主线二：从 CPU 到 GPU 到专用硬件

```
CPU 软件渲染（1992）
  → GPU 固定管线（1996）
    → GPU 可编程着色器（2001）
      → GPU 统一着色器 + 计算着色器（2007）
        → RT Core + Tensor Core + 传统着色器（2018）
```

每一代硬件都在为渲染技术"解锁"新的可能性。**硬件的进步不是渲染技术的"加速器"，而是"使能器"——很多技术在理论上早已存在，只是在等待硬件的到来。**

### 主线三：从手工到自动化

```
手工调参（早期着色器）
  → PBR 材质标准（物理驱动）
    → 程序化生成（自动 LOD、Nanite）
      → AI 驱动（DLSS、帧生成）
        → 生成式引擎（神经世界模型）
```

渲染管线中越来越多的环节被自动化——从材质定义到几何细节，再到超分辨率和帧率。这条主线的终点，可能就是我们在 [Generative Game Engine 系列](../Generative%20Game%20Engine/) 中讨论的生成式引擎。
